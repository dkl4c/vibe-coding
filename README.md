## 📖 简介

这是 mako 设计的**个人学习记录网站模板**，采用日漫樱花风格。

**无需后端、数据库或注册账号** — 所有数据保存在浏览器本地（localStorage），开箱即用。

您可以 Fork 本项目，修改为自己的学习记录网站。

## ✨ 功能特色

### 📚 课程管理
- 展示课程列表，支持添加/编辑/删除课程
- 每个课程有独立的详情页面
- 支持设置课程进度和学习状态

### 📝 课程笔记
- 支持 Markdown 语法（实时预览）
- 按课程分类独立保存
- 支持上传 Markdown/文本文件
- 本地存储，刷新不丢失

### 📋 作业管理
- 按课程分组管理作业
- 设置截止日期
- 标记完成状态
- 带日期的作业自动同步到日程待办

### 📅 日程与课程表
- 待办事项管理（支持截止日期、关联课程）
- 可视化课程表（支持单双周）
- 学期进度自动计算
- 截止日期颜色提示（过期/今天/即将到期）

### 🎨 界面设计
- 樱花粉 + 薄荷绿配色
- 樱花飘落动画背景
- 淡入滚动效果
- 响应式布局（支持手机和电脑）

## 🚀 快速开始

### 1. Fork 本项目
点击 GitHub 右上角的 **Fork** 按钮，将项目复制到您的仓库。

### 2. 克隆到本地
```bash
git clone https://github.com/您的用户名/vibe-coding.git
cd vibe-coding
```

### 3. 本地预览
推荐使用 VS Code 的 Live Server 扩展：
1. 安装 Live Server 扩展
2. 右键点击 `index.html`
3. 选择 "Open with Live Server"

或者直接双击 `index.html` 用浏览器打开（部分功能可能受限）。

### 4. 自定义内容
- 修改 `index.html` 中的个人介绍和课程信息
- 在"帮助"页面设置您的 GitHub 仓库路径
- 上传您的背景图片作为网站背景
- 修改 `courses.js` 中的默认课程数据

### 5. 部署到 GitHub Pages
1. 进入您的仓库 **Settings** > **Pages**
2. Source 选择 **Deploy from a branch**
3. Branch 选择 **main**，文件夹选择 **/ (root)**
4. 点击 **Save**
5. 等待 1-2 分钟，网站即可上线

## 📂 项目结构

```

├── notes.html          # 课程笔记
```

## ⚙️ 自定义设置

在"帮助"页面可以配置：

| 设置项 | 说明 |
|--------|------|
| GitHub 仓库路径 | 上传文件时显示正确的路径提示 |
| 背景图片 | 上传本地图片作为网站背景 |

## 💡 使用技巧

### 笔记
- 选择课程后开始记录
- 使用 Markdown 语法编写
- 点击"💾 保存"按钮保存，或关闭页面时按提示保存

### 作业与待办
- 添加带截止日期的作业时，会自动创建关联待办
- 完成作业或待办任一方，另一方同步完成

### 课程表
- 点击"⚙️ 设置"修改学期开始日期和总周数
- 点击"本周"快速回到当前周
- 单周/双周课程自动根据当前周显示

## 📝 数据存储

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 课程信息 | localStorage | 包括默认课程和用户添加的课程 |
| 笔记内容 | localStorage | 按课程独立保存 |
| 作业记录 | localStorage | 包含标题、截止日期、完成状态 |
| 待办事项 | localStorage | 与作业关联 |
| 课程表 | localStorage | 时间节次和课程项 |
| 自定义设置 | localStorage | GitHub路径、背景图片等 |

**注意**：清除浏览器数据会导致所有本地存储的数据丢失。
